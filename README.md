# 🎉 BINGO LIVE Pro - Multi-Room Edition

ブラウザだけで完結する、リアルタイム同期型のマルチユーザー・ビンゴ抽選システムです。
ルーム名（合言葉）を決めるだけで、複数のグループが同時に別々のビンゴ大会を開催できます。

## 🚀 ライブデモ
**[https://tamachamo.github.io/bingo/](https://tamachamo.github.io/bingo/)**

## ✨ 主な機能

- **マルチプロジェクト対応**: 好きなルーム名を決めるだけで、独立したビンゴゲームを無制限に作成可能。
- **URLクイック招待**: `?room=部屋名` 付きのURLを送るだけで、参加者は入力不要で自動ログイン。
- **P2Pリアルタイム同期**: PeerJS (WebRTC) を活用し、主催者の「数字を引く」操作を全参加者へ即座に反映。
- **ルーム別データ永続化**: ルームごとに抽選履歴を保存。ブラウザを閉じても、その部屋の続きから再開可能（LocalStorage）。
- **演出用シャッフル**: 抽選の瞬間を盛り上げる、緊張感のあるデジタルシャッフルエフェクト。

## 🛠 使い方

### 1. 運営者（ホスト）として部屋を作る
1. トップページで好きな**ルーム名（合言葉）**を入力します。
2. 「運営者として開始」ボタンを押します。
3. 「招待URLをコピー」を押し、LINEやSNSで参加者に共有します。
4. 全員の準備ができたら「数字を引く！」でスタート！

### 2. 参加者（ビューアー）として入る
1. 運営者から送られてきたURLをタップします。
2. 自動的に指定のルームに接続され、同期が開始されます。
   - ※直接トップページから入る場合は、同じルーム名を入力して「参加」してください。

## 🔧 技術スタック

- **Frontend**: HTML5 / CSS3 / JavaScript (Vanilla JS)
- **Communication**: [PeerJS](https://peerjs.com/) (WebRTC)
- **Hosting**: GitHub Pages
- **Storage**: LocalStorage API

## 📝 運用アドバイス

- **ルーム名の重複に注意**: 他のユーザーと同じルーム名にすると接続できない場合があります。イベント名や日付を入れるのがおすすめです（例: `urupuya-2026-party`）。
- **安定した接続**: WebRTCを使用しているため、安定したWi-Fi環境での利用を推奨します。
