# 🎉 BINGO LIVE Pro - Custom Range & Multi-Room

ブラウザだけで完結する、リアルタイム同期型のマルチユーザー・ビンゴ抽選システムです。
ルーム名と最大数字を自由に設定できるため、少人数のパーティーから大規模なイベントまで幅広く対応します。

## 🚀 ライブデモ
**[https://tamachamo.github.io/bingo/](https://tamachamo.github.io/bingo/)**

## ✨ 主な機能

- **最大数字のカスタマイズ**: 抽選範囲を自由に設定可能（デフォルトは75）。1〜999まで、イベントの規模に合わせて調整できます。
- **マルチルーム対応**: 好きなルーム名を決めるだけで、独立したビンゴゲームを無制限に作成・同時開催可能。
- **URLクイック招待**: `?room=部屋名` 付きのURLを送るだけで、参加者は自動ログイン。最大数字の設定も自動で同期されます。
- **P2Pリアルタイム同期**: PeerJS (WebRTC) を活用。主催者の操作を全参加者の画面へ即座に反映します。
- **データ永続化**: ルームごとに「最大数字」と「抽選履歴」をブラウザに保存。再読み込みしても続きから再開できます。

## 🛠 使い方

### 1. 運営者（ホスト）として開始
1. トップページで**ルーム名（合言葉）**を入力します。
2. **最大数字**を入力します（例：100人規模のイベントなら100）。
3. 「運営者として開始」ボタンを押します。
4. 「招待URLをコピー」を押し、参加者に共有します。

### 2. 参加者（ビューアー）として入る
1. 共有されたURLをタップするだけで、自動的に指定のルームに接続・同期されます。
2. 運営者が設定した最大数字の範囲内で、リアルタイムに抽選結果が表示されます。

## 🔧 技術スタック

- **Frontend**: HTML5 / CSS3 / JavaScript
- **Communication**: [PeerJS](https://peerjs.com/) (WebRTC)
- **Hosting**: GitHub Pages
- **Storage**: LocalStorage API

## 📝 運用アドバイス

- **ルーム名の設定**: 他の利用者と重複しないよう、固有の名称（例：`event-2026-0226`）を推奨します。
- **リセット機能**: 新しいゲームを始める際は、運営者画面の「リセット」ボタンで履歴と設定をクリアできます。
